## 2. Handler List

### Sales Order

Schema: `erp_procurement.sales_order` + `erp_procurement.sales_order_line`

| Function | Route | Description |
|---|---|---|
| `createSOHandler` | `POST:/api/procurement/sales-orders` | Create SO. so_number via `generate_doc_number('SO')`. customer_po_number mandatory. Lines: material_id (RM/PM only — validate from material_master.material_type), rate, quantity, gst_rate, discount_pct. net_rate = rate × (1 - discount_pct/100). Status CREATED. |
| `listSOsHandler` | `GET:/api/procurement/sales-orders` | List SOs. Filters: company_id, status, customer_id, date range. |
| `getSOHandler` | `GET:/api/procurement/sales-orders/:id` | Get SO with lines + dispatch status. |
| `updateSOHandler` | `PUT:/api/procurement/sales-orders/:id` | Update CREATED SO only. Not if ISSUED or beyond. |
| `cancelSOHandler` | `POST:/api/procurement/sales-orders/:id/cancel` | Cancel SO. Condition: no stock issued against any line (issued_qty = 0 on all lines). Reason mandatory. Status → CANCELLED. |

### SO Stock Issue + DC + Gate Exit

| Function | Route | Description |
|---|---|---|
| `issueSOStockHandler` | `POST:/api/procurement/sales-orders/:id/issue` | Issue stock for SO line(s). For each line in request: 1) Validate material_type = RM or PM (from material_master). 2) Check stock availability in UNRESTRICTED for issue_storage_location_id. 3) Call post_stock_movement (SALES_ISSUE, OUT, UNRESTRICTED). 4) Auto-create Delivery Challan (dc_type='SALES', so_id). 5) Auto-create Gate Exit Outbound (exit_type='SALES', so_id, dc_id). Update SO line: issued_qty += qty, balance_qty = quantity - issued_qty. Update line_status: PARTIALLY_ISSUED if partial, FULLY_ISSUED if complete. Update SO header status: CREATED → ISSUED (if first issue). |
| `knockOffSOLineHandler` | `POST:/api/procurement/sales-orders/:id/lines/:line_id/knock-off` | Close SO line with reason. balance_qty → 0. line_status → KNOCKED_OFF. If all lines KNOCKED_OFF or FULLY_ISSUED → SO status → CLOSED. |

### Sales Invoice

Schema: `erp_procurement.sales_invoice` + `erp_procurement.sales_invoice_line`

| Function | Route | Description |
|---|---|---|
| `createSalesInvoiceHandler` | `POST:/api/procurement/sales-invoices` | Create Sales Invoice. invoice_number via `generate_invoice_number()` (YYYYMM format). Must reference a dc_id (Delivery Challan). gst_type derived from company.state vs customer.state. Lines loaded from DC lines. |
| `listSalesInvoicesHandler` | `GET:/api/procurement/sales-invoices` | Filters: company_id, customer_id, status, date range. |
| `getSalesInvoiceHandler` | `GET:/api/procurement/sales-invoices/:id` | Get invoice with lines + SO + DC references. |
| `postSalesInvoiceHandler` | `POST:/api/procurement/sales-invoices/:id/post` | Post Sales Invoice. Status DRAFT → POSTED. Records posted_by + posted_at. Compute line totals. Update SO status → INVOICED if all dispatched qty is now invoiced. |

## 3. Sales Invoice Computation (inside `createSalesInvoiceHandler` and `postSalesInvoiceHandler`)

**Per invoice line:**
```
taxable_value = quantity × rate
gst_amount    = taxable_value × gst_rate / 100

if gst_type = 'CGST_SGST':
  cgst_amount = gst_amount / 2
  sgst_amount = gst_amount / 2
  igst_amount = NULL

if gst_type = 'IGST':
  igst_amount = gst_amount
  cgst_amount = NULL
  sgst_amount = NULL

line_total = taxable_value + gst_amount
```

**Header totals:**
```
total_taxable_value = sum(lines.taxable_value)
total_cgst_amount   = sum(lines.cgst_amount)
total_sgst_amount   = sum(lines.sgst_amount)
total_igst_amount   = sum(lines.igst_amount)
total_gst_amount    = sum(lines.cgst + sgst + igst)
total_invoice_value = total_taxable_value + total_gst_amount
```

**GST type derivation:**
```
Fetch company.state and customer.state from erp_master
if company.state = customer.state: gst_type = 'CGST_SGST'
else: gst_type = 'IGST'
```

## 4. Business Rules

| Rule | Detail |
|---|---|
| RM/PM only | Validate material_type IN ('RM','PM') before allowing in SO line. 400 if FG or other type. |
| No manual DC | DC auto-generated by issueSOStockHandler. Cannot manually create Sales DC. |
| No manual Gate Exit | Gate Exit auto-generated on issue. |
| SALES_ISSUE movement | Must exist in erp_inventory.movement_type_master. Direction = OUT. Stock type = UNRESTRICTED. |
| Invoice number format | `generate_invoice_number()` — YYYYMM + 6-digit incremental. Global counter. |
| gst_type no NONE | Sales invoices always have GST (CGST_SGST or IGST). Never NONE. |
| No approval for issue | Stock issue requires no approval. Stores user can issue directly. |
| SO status auto-update | After posting all invoices for dispatched qty → SO status = INVOICED. After knock-off = CLOSED. |
